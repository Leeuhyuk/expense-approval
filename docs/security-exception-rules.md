# Security And Exception Rules

작성일: 2026-07-05

이 문서는 비밀번호, 저장 중 이탈, 관리자 수동 복구, 데이터 정합성 검증의 1차 기준이다.

## 비밀번호 정책

- 최소 길이는 12자다.
- 영문 대문자, 영문 소문자, 숫자, 특수문자를 각각 1개 이상 포함한다.
- 비밀번호 최대 사용 기간은 90일이다.
- 로그인 실패 5회 이후 계정을 잠그고 15분 실패 window가 지나면 재시도할 수 있다. 운영 환경에서는 `LOGIN_FAILURE_LOCK_THRESHOLD`, `LOGIN_FAILURE_WINDOW_MINUTES`로 조정한다.
- 90일 동안 로그인하지 않은 계정은 휴면 계정으로 차단하고 관리자 재활성화 절차를 요구한다. 운영 환경에서는 `DORMANT_ACCOUNT_DAYS`로 조정한다.
- 유휴 세션은 30분 후 만료한다.

## 저장 중 화면 이탈

- 저장 API 호출 중에는 라우트 이동과 브라우저 이탈을 막는 것을 원칙으로 한다.
- 변경 중인 draft가 있고 저장 중이 아니면 이탈 확인을 띄운다.
- 임시 저장 가능한 입력은 30초 주기로 draft autosave 대상이다.
- 저장 실패 후 이동하면 마지막 성공 저장 시점의 데이터만 유효한 것으로 본다.

## 관리자 수동 복구

- 관리자 수동 복구는 `system:manage` 권한을 가진 관리자가 요청하고, 다른 `system:manage` 관리자가 2차 승인해야 적용된다.
- 복구 요청, 승인, 반려에는 사유와 idempotencyKey가 필수다.
- 지급 완료 건 복구는 `/api/operations/manual-recoveries` 요청과 `/approve` 2차 승인을 거쳐야 하며 요청자 본인 승인은 차단한다.
- 복구 대상은 rowVersion, 기존 상태, 변경 후 상태, 요청자, 승인자, requestId를 `manual_recovery` 감사 로그에 함께 기록한다.

## 데이터 정합성 검증

- 상태 변경 API는 rowVersion을 필수로 검증한다.
- 승인, 반려, 보류, 지급 실행 같은 액션 API는 idempotencyKey를 필수로 검증한다.
- 지급 건은 승인 완료 요청과 금액, 거래처, 승인번호가 일치해야 한다.
- 예산 사용액은 일 1회 승인/지급 데이터와 대사한다.
- 정합성 불일치는 관리자 알림과 감사 로그 대상으로 본다.
