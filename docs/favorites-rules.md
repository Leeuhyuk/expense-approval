# Favorites Rules

작성일: 2026-07-05

이 문서는 즐겨찾기 화면의 1차 구현 기준이다.

## 데이터 구성

- 즐겨찾기는 메뉴, 필터, 보고서 유형을 가진다.
- 각 항목은 제목, 설명, 최근 사용 시각, 소유자, 상태, 연결 필터, 사용 횟수, 공유 범위를 가진다.
- 자주 쓰는 메뉴, 저장된 필터, 최근 사용 목록은 같은 즐겨찾기 상태에서 파생해 표시한다.

## 사용자별 저장

- 현재 구현은 `FavoriteItem` backend API를 기준으로 조회, 추가, 순서 편집, 삭제, 복구, 사용자 저장을 처리한다.
- 브라우저 전용 `erp-favorites:{userId}` 저장 경로는 사용하지 않는다.
- 사용자별 저장은 현재 순서, 삭제 상태, 추가된 바로가기, 연결 필터, 공유 범위를 함께 보존해야 한다.

## 바로가기 추가와 중복 방지

- 바로가기 추가는 기본 예시인 `예산 초과 알림`을 생성한다.
- 같은 제목의 즐겨찾기가 이미 있으면 새 항목을 만들지 않고 기존 항목을 선택한다.
- 중복 판단은 1차 구현에서는 제목 기준이며, API 단계에서는 대상 화면과 필터 조건까지 포함한다.

## 순서와 삭제

- 순서 편집은 선택된 항목을 한 단계 위로 이동한다.
- 선택 항목이 첫 번째이면 맨 뒤로 이동한다.
- 삭제는 선택된 즐겨찾기를 backend에서 비활성화하고 남은 첫 항목을 선택한다.
- undo 복구는 같은 항목을 다시 활성화한다.

## 연결 필터

- 연결 필터는 화면별 필터 조건을 사용자에게 보이는 태그로 표시한다.
- 필터 유형 즐겨찾기는 최소 1개 이상의 연결 필터 태그를 가져야 한다.
- 메뉴와 보고서 유형도 저장된 조건이 있으면 상세 패널에 연결 필터를 표시한다.

## 비활성 메뉴 처리

- 비활성 메뉴는 최근 사용 목록과 상세 패널에서 조회할 수 있다.
- 비활성 메뉴는 열기와 신규 바로가기 추가 대상에서 제외한다.
- 비활성 메뉴는 사용자가 정리할 수 있도록 삭제는 허용한다.
