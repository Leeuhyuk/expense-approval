# Settings Rules

작성일: 2026-07-05

이 문서는 시스템 설정 화면의 1차 구현 기준이다.

## 결재 정책

- 결재 정책 저장은 현재 승인 한도와 결재선 규칙을 함께 저장한 것으로 본다.
- 승인 한도 구간은 금액 범위, 결재 단계, 필수 승인자 수, 상태를 가진다.
- 승인 한도 구간은 추가, 수정, 삭제할 수 있지만 최소 1개 구간은 유지한다.
- 결재선 규칙은 기본 결재선 모드, 병렬 결재, 대리 결재, 부재 예외, 거래처 예외 결재선으로 구성한다.

## 권한과 사용자

- 권한 그룹은 결제 요청, 승인 관리, 지급 관리, 예산 관리, 보고서, 시스템 설정 화면별 접근 권한을 가진다.
- 권한 그룹 활성 상태가 비활성이면 신규 사용자 배정 대상에서 제외하는 것이 원칙이다.
- 사용자 권한 추가는 사용자명, 권한 그룹, 역할을 함께 저장한다.
- 권한 변경은 감사 목적의 변경 이력에 작업자와 변경 대상을 남긴다.

## 알림과 외부 연동

- 알림 설정은 승인 대기, 지급 예정, 정책 변경 알림의 연동 여부를 관리한다.
- 외부 연동 설정은 회계 시스템, 세금계산서 수집, 은행 계좌 검증을 기본 구조로 둔다.
- 외부 연동 상태는 `연동`, `대기`, `점검` 중 하나로 표시한다.

## 적용 범위

- 신규 결제 요청, 신규 결재선 선택, 권한 그룹 변경은 설정 저장 즉시 적용한다.
- 진행 중 결재 건은 생성 당시 결재선 스냅샷을 유지한다.
- 진행 중 결재 건이 보류 또는 반려 후 재상신되면 신규 정책을 다시 평가한다.
- 기존 지급 완료, 승인 완료 건에는 정책 변경을 소급 적용하지 않는다.

## 변경 이력

- 정책, 권한, 사용자, 알림, 연동 변경은 변경 이력에 기록한다.
- 변경 이력은 시간, 작업자, 설명, 유형을 포함한다.
- 1차 구현은 브라우저 상태 기반 mock 이력이며, 백엔드 연결 단계에서 감사 로그 API로 이관한다.
