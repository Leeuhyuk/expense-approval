# Frontend Structure Guidelines

작성일: 2026-07-04

이 문서는 현재 화면 디자인을 유지하면서 프론트 기능을 공통화하는 1차 기준이다.

## 공통 테이블 적용 범위

1차 공통화 대상:

- 결제 요청 목록
- 승인 관리 목록
- 지급 관리 목록

공통 컨트롤러: `useManagedTable`

제공 기능:

- 검색, 상태 필터, 페이지네이션
- 행 선택과 다중 선택
- 정렬
- 목록 새로고침
- 로딩, 오류, 빈 데이터 상태
- 작업 중 중복 클릭 방지
- 성공/실패 메시지

예산, 거래처, 보고서, 즐겨찾기 테이블은 화면 구성이 더 고정적이므로 2차 범위에서 `useManagedTable` 또는 별도 `useStaticManagementTable`로 편입한다.

## 공통 필터 적용 범위

1차 공통화 대상:

- 상태 필터: 결제 요청, 승인 관리, 지급 관리
- 검색어: 상단 검색 입력
- 페이지 크기: 10, 20, 50 순환

2차 공통화 대상:

- 날짜 범위
- 부서
- 거래처
- 보고서 유형
- 계좌 확인 상태

필터는 UI 표시값을 유지하되 API에는 `filters.{field}` 형태로 전달한다.

## 공통 우측 상세 패널 적용 범위

1차 공통 패턴:

- 헤더: 제목, 상태 배지, 닫기 버튼
- 본문: 주요 필드 `dl`, 첨부/이력/흐름 카드
- 하단 액션: 상태와 권한을 함께 검증한 버튼
- 메시지: 성공/실패/검증 결과를 `panel-action-message`로 표시

개별 화면별 차이:

- 결제 요청: 입력/첨부/예산/결재선
- 승인 관리: 첨부/결재선/승인 액션
- 지급 관리: 계좌/일정/지급 액션

## 상태 배지 표시 규칙

| 업무 | 상태 | 표시 톤 |
| --- | --- | --- |
| 결제 요청 | 임시 저장 | 중립 |
| 결제 요청 | 제출, 승인 대기 | 진행 |
| 결제 요청 | 승인 완료 | 완료 |
| 결제 요청 | 반려 | 위험 |
| 승인 | 승인 대기, 승인 진행 중 | 진행 |
| 승인 | 승인 완료 | 완료 |
| 승인 | 반려 | 위험 |
| 승인 | 보류 | 주의 |
| 지급 | 지급 예정 | 예정 |
| 지급 | 오늘 지급 | 강조 |
| 지급 | 지급 완료 | 완료 |
| 지급 | 오류 | 위험 |
| 지급 | 보류 | 주의 |

상태 배지는 값 자체를 숨기지 않고, 색상은 보조 신호로만 사용한다.

## 금액과 날짜 포맷

공통 유틸: `src/domain/formatters.ts`

- 금액 표시: `formatCurrencyWon(number)` → `1,000 원`
- 날짜 표시: `formatDateIso(date)` → `YYYY-MM-DD`
- 정렬 비교: 날짜, 금액, 문자열을 같은 테이블 정렬 함수에서 처리

## 긴 텍스트와 큰 금액 표시

- 테이블 헤더는 `table-sort-button`으로 내부 텍스트와 정렬 아이콘이 같은 줄에 유지된다.
- 알림과 보고서처럼 긴 설명은 두 줄 제한 또는 테이블 셀 폭 기준으로 말줄임한다.
- 큰 금액은 쉼표 포함 원화 문자열로 표시하고 정렬 시 숫자로 파싱한다.

## 빈 데이터, 로딩, 오류 상태

공통 상태 행: `TableStateRow`

- 로딩: `목록을 불러오는 중입니다.`
- 오류: 오류 메시지와 `다시 시도` 버튼
- 빈 데이터: `조건에 맞는 데이터가 없습니다.`

목록 상태 행은 테이블 구조를 유지하기 위해 `tbody` 안에서 전체 컬럼을 `colSpan`으로 차지한다.

## 결제 요청 입력 검증

1차 적용 범위: 결제 요청 우측 상세 패널

- 임시 저장: 선택된 요청의 거래처, 부서, 요청일, 금액, 요청 사유를 현재 draft 값으로 저장한다.
- 제출: 거래처, 부서, 요청일, 1원 이상 금액, 요청 사유, 증빙 파일 1개 이상, 예산 잔액 초과 여부를 검증한다.
- 금액 입력: 숫자만 허용하고 저장/제출 시 `formatCurrencyWon`으로 원화 표시를 맞춘다.
- 예산 확인: 선택 부서의 mock 예산 잔액과 요청 금액을 비교해 집행 후 잔액을 표시한다.
- 결재선 표시: 요청 금액 기준으로 기본 결재선 또는 고액 결재선을 예상 표시한다.
- 제출 이후: 상태가 `제출`로 변경되면 입력과 첨부 변경을 비활성화한다.

## 승인 처리 입력 규칙

1차 적용 범위: 승인 관리 목록과 우측 상세 패널

- 내 요청: 현재 사용자가 포함된 결재선 기준으로 승인 목록을 필터링한다.
- 일괄 승인: 선택된 행 중 `승인 대기`, `승인 진행 중` 상태만 승인 완료로 변경한다.
- 반려/보류: 사유 입력을 필수로 검증하고, 처리 사유와 처리 이력을 mock 행 데이터에 저장한다.
- 결재선 현황: 승인 상태값에 따라 진행, 완료, 반려, 보류 상태를 우측 패널에 표시한다.
- 첨부 파일: 요청번호, 거래처, 요청일, 금액 기준으로 세금계산서, 견적서, 고액 계약서를 동적 표시한다.
- 결재 순서: 현재 활성 결재자가 현재 사용자일 때만 승인, 반려, 보류 처리를 허용한다.
- 승인 처리: 다음 결재자가 남아 있으면 `승인 진행 중`, 마지막 결재자이면 `승인 완료`로 전환한다.

## 지급 처리 입력 규칙

1차 적용 범위: 지급 관리 목록과 우측 상세 패널

- 일괄 지급: 선택된 행 중 지급 상태가 `지급 예정`, `오늘 지급`, `보류`이고 계좌확인이 `확인 완료`인 건만 지급 완료 처리한다.
- 계좌 확인: `확인 대기`, `계좌 불일치` 건은 우측 패널에서 계좌 재확인을 통해 `확인 완료`로 전환한다.
- 지급 일정: 우측 패널의 지급 예정일 입력값이 목록의 `지급예정일`과 연동된다. 지급 완료 건은 변경하지 않는다.
- 보류 처리: 보류 사유를 필수로 검증하고 `지급 보류 사유`, `지급 이력`에 저장한다.
- 승인 연결: 지급 건의 `승인번호`로 승인 요청의 요청일, 부서, 요청자, 결재상태를 조회해 표시한다.
- 오류 재처리: 오류 건은 계좌 재확인 후 `지급 예정`으로 되돌리고, 이후 지급 실행을 재시도한다.

## 예산 관리 입력 규칙

1차 적용 범위: 예산 관리 목록, 항목 차트, 우측 상세 패널

- 기간 필터: 회계 기간 표시를 순환하고 우측 상세 패널의 회계 기간과 동기화한다.
- 부서 필터: 선택 부서 기준으로 사용 현황 목록과 상세 패널을 갱신한다.
- 상태 필터: `정상`, `주의`, `초과` 상태 기준으로 부서별 예산 목록을 필터링한다.
- 항목 필터: 예산 항목별 차트의 표시 항목을 갱신한다.
- 예산 조정: 선택 부서의 배정 예산과 잔액을 조정하고 조정 이력을 우측 패널에 표시한다.
- 초과 위험: 사용률 80% 이상은 주의, 100% 이상은 초과로 판정해 경고 목록에 표시한다.

상세 계산 기준은 `docs/budget-rules.md`를 따른다.

## 거래처 관리 입력 규칙

1차 적용 범위: 거래처 관리 목록과 우측 상세 패널

- 검색: 거래처명, 사업자번호, 담당자, 은행 문자열을 대상으로 필터링한다.
- 상태 필터: `활성`, `비활성` 기준으로 목록을 필터링한다.
- 계좌 필터: `확인 완료`, `검증 대기`, `계좌 불일치`, `비활성` 기준으로 목록을 필터링한다.
- 거래처 추가: 신규 거래처를 `활성`, `검증 대기` 상태로 생성하고 우측 패널에서 즉시 편집한다.
- 거래처 수정: 거래처명, 사업자번호, 담당자, 은행, 계좌번호, 계좌확인 상태, 세금계산서 정보를 저장한다.
- 중복 검증: 저장 시 사업자번호와 계좌번호가 다른 거래처와 중복되면 저장하지 않는다.
- 계좌 재확인: `검증 대기`, `계좌 불일치`, `비활성` 계좌를 `확인 완료`로 갱신한다.
- 비활성화: 거래처 상태와 계좌확인 상태를 `비활성`으로 변경한다.
- 결제 요청 제한: 결제 요청 거래처 선택 목록은 활성 거래처만 포함한다.
- 최근 지급 요청: 거래처명 기준으로 지급 관리 데이터를 조회해 우측 패널에 표시한다.

## 보고서 관리 입력 규칙

1차 적용 범위: 보고서 화면의 필터, 차트, 목록, 사이드 패널

- 유형 필터: `종합`, `지급`, `승인`, `예산` 유형에 맞춰 보고서 목록을 필터링한다.
- 보고서 생성: 현재 기간과 유형으로 새 보고서를 생성해 목록과 저장된 보고서에 추가한다.
- 월별 지급 추이: 지급 관리 mock 데이터의 금액 합계를 기반으로 월별 추이 값을 계산한다.
- 부서별 지출: 승인 관리 mock 데이터의 부서별 금액 합계를 기준으로 막대 차트를 표시한다.
- 승인 상태: 승인 관리 mock 데이터의 결재상태 개수를 기준으로 상태 차트를 표시한다.
- 예약 발송: backend `ReportSchedule` API로 예약 목록을 조회하고, 추가/수정/중지 버튼을 저장한다.
- 내보내기: 선택된 보고서를 `ReportRun` 다운로드 API에 요청하고 backend가 생성한 CSV/PDF 파일 payload를 저장한다.

상세 기준은 `docs/report-rules.md`를 따른다.

## 시스템 설정 입력 규칙

1차 적용 범위: 시스템 설정 화면의 결재 정책, 사용자 권한, 알림, 외부 연동 설정

- 결재 정책 저장: 승인 한도, 결재선 규칙, 적용 범위 정의를 현재 설정 상태로 저장하고 변경 이력에 기록한다.
- 승인 한도: 금액 구간을 추가, 수정, 삭제할 수 있으며 최소 1개 구간은 유지한다.
- 결재선 규칙: 기본 결재선 모드와 병렬 결재, 대리 결재, 부재 예외, 거래처 예외 결재선을 저장한다.
- 권한 그룹: 권한 그룹을 추가하고 화면별 권한과 활성 상태를 변경한다.
- 사용자 권한: 사용자명, 권한 그룹, 역할을 입력해 사용자 권한 배정을 추가한다.
- 변경 이력: 정책, 권한, 사용자, 알림, 연동 설정 변경은 시간, 작업자, 설명, 유형으로 표시한다.
- 알림 설정: 승인 대기, 지급 예정, 정책 변경 알림의 연동 여부를 저장한다.
- 외부 연동: 회계 시스템, 세금계산서 수집, 은행 계좌 검증 연동 상태를 관리한다.

상세 기준은 `docs/settings-rules.md`를 따른다.

## 즐겨찾기 입력 규칙

1차 적용 범위: 즐겨찾기 화면의 자주 쓰는 메뉴, 저장된 필터, 최근 사용 목록, 상세 패널

- 자주 쓰는 메뉴: 활성 즐겨찾기 중 메뉴와 보고서 유형을 카드로 표시한다.
- 저장된 필터: 필터 유형 즐겨찾기를 별도 카드 목록으로 표시하고 연결 필터 태그를 노출한다.
- 최근 사용: 유형 필터와 비활성 필터에 따라 같은 즐겨찾기 데이터를 테이블로 표시한다.
- 바로가기 추가: 사용자별 즐겨찾기에 예산 초과 알림 바로가기를 추가한다.
- 순서 편집: 선택된 즐겨찾기의 순서를 위로 이동하고, 첫 항목이면 맨 뒤로 보낸다.
- 삭제: 선택된 즐겨찾기를 목록에서 제거하고 다음 항목을 선택한다.
- 연결 필터: 상세 패널에 선택 항목의 연결 필터 태그를 표시한다.
- 사용자별 저장: `FavoriteItem` backend API에 현재 즐겨찾기 상태, 순서, 필터, 공유 범위를 저장한다.
- 중복 방지: 같은 제목의 바로가기가 이미 있으면 새 항목을 만들지 않고 기존 항목을 선택한다.
- 비활성 메뉴: 열기와 신규 바로가기 추가는 차단하고, 조회와 삭제만 허용한다.

상세 기준은 `docs/favorites-rules.md`를 따른다.
